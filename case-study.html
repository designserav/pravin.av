    • Goal

    Design a platform that:

    • Helps students find affordable tutors easily
    • Enables students to earn by teaching
    • Provides simple scheduling and payments
    • Creates a smooth learning experience

    The Problem

    Students face two major challenges:

    • Difficulty finding affordable and personalized tutors
    • Lack of earning opportunities while studying

    Tutors also struggle with:

    • Finding consistent students
    • Managing schedules and payments
    👉 Existing platforms are expensive, complex, and not student-friendly.

    Research Insights

    • Students prefer low-cost and flexible tutoring
    • Tutors want easy onboarding and quick income
    • Users need simple UI (not complex dashboards)
    • Trust and clarity are very important
    👉 Insight: Simplicity and affordability are key to success

    Solution

    Designed Slate platform that:

    • Connects students with peer tutors
    • Allows users to both learn and earn
    • Provides live sessions and recorded content
    • Includes simple booking and payment flow

    User Flow

    Focus on minimal steps and smooth experience:

    1. User signs up 2. Chooses Learn or Earn 3. Browses or creates courses 4. Books session 5. Attends live class

    Usability Improvements

    • Reduced booking steps
    • Improved button visibility
    • Simplified onboarding

    UI Design

    Key Features:

    • Live session interface
    • Simple booking flow

    Design Decisions:

    • Strong typography

    Design System

    • Consistent spacing system
    • Reusable components
    • Typography hierarchy
    • Color system

    Wireframes

    Created low fidelity wireframes focused on:

  • Simple navigation
  • Clear hierarchy
  • Easy user actions

    • Impact

    • Improved user flow clarity
    • Reduced friction in booking
    • Increased engagement

    Learnings

    • Simplicity is more powerful than features
    • Users prefer clarity over complexity
    • Early testing saves time
